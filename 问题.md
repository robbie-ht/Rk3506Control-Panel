# 项目问题整理

## 1. 严重问题：resources/power.png 文件缺失

**问题描述：**
- `resources.qrc` 文件中引用了 `resources/power.png`
- 但 `resources/` 目录是空的，没有 `power.png` 文件 这里我已经手动修改好了
- `power.png` 实际存在于 `build_x86/` 目录中

**影响：**
- 编译时会失败，因为资源文件不存在
- 电源按钮无法显示图片（会回退到 QPainter 绘制）

**解决方案：**
- 将 `build_x86/power.png` 复制到 `resources/` 目录
- 或者从其他位置获取 power.png 文件

## 2. Git 状态问题：未提交的更改

**问题描述：**
- 有大量文件被修改但未提交
- 有文件被删除但未提交
- 有新文件/目录未被跟踪

**具体状态：**
```

- LyraShell.pro
- helloworld/helloworld.cpp
- lyramainwindow.cpp
- resources.qrc
- sidebar/sidebar.cpp
- sidebar/sidebaritem.cpp

已删除（未暂存）：
- helloworld/README.md
- helloworld/build/ 目录下的文件
- mainwindow.cpp
- mainwindow.h
- plugins/libhelloworld.so

未跟踪的文件：
- build/

- helloworld/plugins/

```

**影响：**
- 代码版本不一致
- 难以追踪更改历史

**解决方案：**
- 提交当前更改：`git add . && git commit -m "描述"`   这个暂时不要管因为你现在的版本还不是完好的版本
- 添加 `.gitignore` 文件忽略 build 目录

## 3. 目录结构问题：build 目录被跟踪

**问题描述：**
- `build/` 和 `build_x86/` 目录包含编译产物
- 这些目录不应该被 git 跟踪

**影响：**
- 仓库体积增大
- 编译产物污染版本控制

**解决方案：**
创建 `.gitignore` 文件：
```
# Build directories
build/
build_x86/
*.o
*.so
*.moc
moc_*.cpp
qrc_*.cpp
Makefile
.qmake.stash

# Qt generated files
*.pro.user
```

## 建议的 Git 操作顺序

```bash
# 1. 添加 .gitignore
echo -e "build/\nbuild_x86/\n*.o\n*.so\n*.moc\nmoc_*.cpp\nqrc_*.cpp\nMakefile\n.qmake.stash" > .gitignore

# 2. 复制 power.png 到正确位置
cp build_x86/power.png resources/

# 3. 添加所有更改
git add .

# 4. 提交更改
git commit -m "重构：主页改为系统信息显示，更新主题样式，修复资源文件问题"

# 5. 推送到远程仓库
git push origin main
```
